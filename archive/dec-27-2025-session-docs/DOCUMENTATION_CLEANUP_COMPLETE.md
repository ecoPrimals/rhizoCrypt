## 📝 Documentation Guidelines

### File Naming
```
Root Directory:
- Use UPPERCASE_WITH_UNDERSCORES.md
- Descriptive names (README.md, STATUS.md, START_HERE.md)
- Avoid generic names (DOC1.md, STATUS2.md)

Subdirectories:
- Use lowercase-with-hyphens.md
- Follow directory conventions
```

### Content Structure
```
1. Clear title and date
2. Status/summary section
3. Table of contents (if >500 lines)
4. Logical sections with headers
5. Code examples with syntax highlighting
6. Cross-references to related docs
7. Footer with metadata
```

### Maintenance Schedule
```
Daily:    STATUS.md (during active development)
Weekly:   Update metrics and progress
Monthly:  Review and archive interim docs
Quarter:  Complete documentation review
Release:  Update CHANGELOG.md, README.md
```
